# 計算ゲーム プロトタイプ（フロントエンド単体）

概要
- 問題形式：5つの2桁の数（10〜99）に + / - をランダムに挟んだ式を出題（例: 73 + 28 - 11 + 54 - 20）。
- ルール：ゲーム開始でタイマー（デフォルト5分）が動き、制限時間内は問題を次々出題。回答は無制限（時間内に何問でも解答可）。時間切れでゲーム終了。
- 記録：各問題ごとに提示時刻・解答時刻・所要時間・回答・正誤・正解を記録。ゲーム単位で統計を保存。
- 永続化：localStorage に保存（キー: calc_games_v1）。管理者は CSV で全ゲームをエクスポート可能。
- BGM：管理者モードで BGM の URL を指定可能。プレイヤーは有効化して再生できる（ブラウザポリシーにより自動再生は制限される場合があります）。
- このプロトタイプはクライアント単体（サーバ不使用）です。配布者による「一括管理（中央収集）」は B フェーズで Firebase 等のバックエンドを導入して実現します。

使い方
1. 提供ファイル（index.html, styles.css, script.js, README.md）を同じフォルダに置く
2. index.html をブラウザで開く
3. 必要なら「管理者モード」ボタンを押し、任意のパスワードを入力して管理者パネルを開く
   - 管理者パネルで BGM の URL を保存すると、プレイヤー側で BGM を有効にした時に再生されます（保存は localStorage）
4. 「ゲーム開始」を押してプレイ。回答を入力して Enter または 送信ボタンで送る
5. 時間切れでゲームが自動終了し、結果が localStorage に保存されます
6. 「全ゲームをCSVでエクスポート」から全データをファイルでダウンロードできます（配布者はこのCSVを回収して一括管理できます）

注意事項
- 管理者パスワードはプロトタイプ内でのローカル保護にすぎません。実運用での管理者認証やデータ管理はサーバサイド（例：Firebase Auth + Firestore）に切り替えてください。
- BGM の著作権は管理者側で適切に管理してください。
- ブラウザの自動再生ポリシーにより、最初のユーザー操作がないと音声再生がブロックされる場合があります（プレイ開始や BGM 有効化の操作がトリガーとなります）。

拡張の次の段階（B フェーズ候補）
- Firebase 等のバックエンドを追加して、参加者データを中央で自動収集・管理（管理者画面で一覧・検索・CSVエクスポート）
- 管理者がアップロードした BGM を Cloud Storage に入れて配信
- 認証（メール／SSO）・権限管理（管理者のみ閲覧）・監査ログ
- UI/UX のブラッシュアップ（デザインフリーズ）およびモバイル最適化（PWA またはネイティブ）
